## 摘要

DFE是接收端基于判决结果的非线性反馈均衡器。它利用已经判决出的历史码元估计这些码元对当前采样点造成的拖尾干扰，并在当前判决前减去这部分post-cursor ISI。DFE关注采样时刻的电平是否正确，而不是恢复完整连续波形。

## 关键词

SerDes, DFE, 判决反馈均衡, Post-cursor ISI, 非线性均衡, 错误传播, 采样点, CDR

## 核心理解

- DFE只能使用过去已经判决出的码元，因此主要处理post-cursor ISI，不能直接处理pre-cursor。
- 反馈输入是判决后的0/1或符号值，所以不会像线性高频提升那样直接放大输入噪声。
- DFE的优势是采样点处抵消能力强，代价是可能出现错误传播：一次判决错误会反馈影响后续若干bit。
- DFE和CDR关系紧密：均衡后的判决质量会影响相位检测，采样点移动又会影响DFE抽头收敛和判决可靠性。

## 1. DFE 要解决的问题：过去码元的拖尾影响当前判决

高速 SerDes 信道通常是低通特性的，一个码元发出去以后，不会只影响自己的采样点，而是会拖到后面几个 UI。

假设发送的是：

```text
a[n] = 当前码元
a[n-1] = 前一个码元
a[n-2] = 前两个码元
```

经过信道后，在当前采样点看到的电压可以写成：

```text
y[n] = h0·a[n] + h1·a[n-1] + h2·a[n-2] + h3·a[n-3] + noise
```

其中：

```text
h0·a[n]
```

是当前码元本身，叫 **main cursor**。

而：

```text
h1·a[n-1] + h2·a[n-2] + h3·a[n-3]
```

就是过去码元拖尾叠加到当前采样点上的干扰，也就是 **post-cursor ISI**。

所以对于当前采样点来说，DFE 要做的是：

```text
把过去已经判决出来的数据拿回来，估计它们对当前采样点造成了多少拖尾，然后减掉。
```

## 2. DFE 的基本结构

DFE 的结构可以理解成：

```text
接收信号 y[n]
      |
      v
   减法器  <------ 反馈滤波器 <------ 已判决数据 â[n-1], â[n-2], â[n-3]
      |
      v
   判决器 Slicer
      |
      v
   输出 â[n]
```

数学上可以写成：

```text
z[n] = y[n] - b1·â[n-1] - b2·â[n-2] - b3·â[n-3]
```

然后再对 `z[n]` 进行判决：

```text
â[n] = slicer(z[n])
```

如果反馈系数满足：

```text
b1 ≈ h1
b2 ≈ h2
b3 ≈ h3
```

那么：

```text
z[n] ≈ h0·a[n] + noise
```

也就是说，过去码元造成的拖尾被抵消掉了，当前判决点就变干净了。

## 3. 举一个具体例子

假设信道冲激响应在采样点上的值是：

```text
h0 = 1
h1 = 0.4
h2 = 0.2
h3 = 0.1
```

也就是说，发送一个单独的 1，接收端采样可能看到：

```text
1, 0.4, 0.2, 0.1
```

这表示第一个码元本身是 1，但它会继续拖尾到后面三个采样点。

假设当前发送数据是：

```text
a[n-3] = 1
a[n-2] = 1
a[n-1] = 1
a[n]   = -1
```

那么当前采样点接收到的是：

```text
y[n] = 1×(-1) + 0.4×1 + 0.2×1 + 0.1×1
     = -1 + 0.7
     = -0.3
```

本来当前码元是 `-1`，但由于前三个码元都是 `1`，它们的拖尾把当前采样点抬高到了 `-0.3`。

如果噪声再稍微大一点，就可能被误判成 `1`。

DFE 会根据已经判决出来的过去数据，减掉：

```text
0.4×â[n-1] + 0.2×â[n-2] + 0.1×â[n-3]
```

如果过去判决正确，那么：

```text
z[n] = y[n] - 0.4×1 - 0.2×1 - 0.1×1
     = -0.3 - 0.7
     = -1
```

这样当前码元又回到了理想的 `-1` 附近，判决余量就明显变大。

## 4. 为什么说 DFE 没有 Pre-cursor Taps？

这个地方很关键。

从**信道冲激响应**角度看：

```text
        pre-cursor     main cursor       post-cursor
            |              |                  |
采样点:    -1              0                 +1, +2, +3
```

DFE 的反馈输入来自已经判决过的历史数据：

```text
â[n-1], â[n-2], â[n-3] ...
```

也就是说，DFE 只能利用过去已经知道的码元。

所以它只能抵消：

```text
过去码元拖尾到当前采样点的影响
```

也就是 post-cursor ISI。

但是 pre-cursor ISI 对当前码元来说，等价于：

```text
未来码元提前影响当前采样点
```

如果要抵消 pre-cursor，就需要知道：

```text
â[n+1], â[n+2]
```

可是这些未来码元还没有判决出来，DFE 无法使用它们。

所以严格说：

```text
DFE 的反馈部分只能消除 post-cursor ISI，不能消除 pre-cursor ISI。
```

pre-cursor 通常要靠：

```text
TX FFE
RX FFE
CTLE
采样相位调整
```

来处理。

## 5. 为什么 DFE 不会像 CTLE/FFE 那样放大高频噪声？

CTLE 和 FFE 都属于前向线性均衡。它们直接处理接收到的模拟波形或者采样数据。

比如信道高频损耗严重，CTLE 会增强高频：

```text
高频信号被增强
高频噪声也被增强
```

这就是 CTLE/FFE 的典型副作用。

但 DFE 不一样。DFE 的反馈输入不是原始模拟波形，而是已经判决后的数字结果：

```text
â[n-1] = 1 或 -1
```

这些数据经过判决后，幅度噪声已经被“硬判决”去掉了。

所以 DFE 反馈支路里产生的是一个“干净的 ISI 估计值”：

```text
b1·â[n-1] + b2·â[n-2] + ...
```

它减掉的是过去码元造成的确定性拖尾，而不是对接收波形做高频增强。

因此可以说：

```text
DFE 消除 ISI，但不会主动放大输入端的高频噪声。
```

不过要注意，不能说“DFE 判决时完全消除了噪声”。更准确的说法是：

```text
如果当前判决是正确的，那么反馈数据可以看作无幅度噪声的理想符号；
但如果判决错误，错误也会进入反馈路径，造成 error propagation。
```

也就是 DFE 有一个问题：**错误传播**。

例如本来 `â[n-1]` 应该是 `1`，但被误判成 `-1`，那么 DFE 在下一拍会用错误的数据去抵消 ISI，反而会给当前采样点引入新的误差。

## 6. 为什么说 DFE 是非线性均衡器？

线性均衡器的输出一般可以写成：

```text
输出 = 输入样本的线性加权和
```

比如 FFE：

```text
z[n] = c-1·y[n+1] + c0·y[n] + c1·y[n-1] + c2·y[n-2]
```

它只是在做线性滤波。

但 DFE 中间有一个判决器：

```text
â[n] = slicer(z[n])
```

`slicer` 是非线性的，因为它会把连续电压值强行映射成离散符号。

例如：

```text
+0.73 -> 1
+0.12 -> 1
-0.08 -> -1
-0.95 -> -1
```

因此 DFE 的反馈路径是：

```text
连续信号 -> 判决成离散符号 -> 再反馈抵消
```

由于中间包含判决器，所以 DFE 是非线性均衡器。

## 7. 图中“只关心采样时刻”的含义

你提到的图，本质上是在说明：

```text
DFE 不要求整个波形在所有时间点都变得理想；
它只要求在判决采样点处 ISI 最小。
```

比如原始信道响应可能是：

```text
采样点:  0UI   1UI   2UI   3UI
电平:    1    0.4   0.2   0.1
```

DFE 通过反馈抵消，希望在采样点处变成：

```text
采样点:  0UI   1UI   2UI   3UI
电平:    1     0     0     0
```

注意，这里的重点是：

```text
在 1UI、2UI、3UI 这些采样时刻为 0
```

而不是要求两个采样点之间的连续波形也完全为 0。

所以 DFE 后的眼图有一个典型特点：

```text
眼图可能只在采样中心附近明显张开；
离采样点较远的位置，波形仍然可能很乱。
```

这和 CTLE/FFE 不一样。CTLE/FFE 是前向滤波，会改变整个波形形状；DFE 则主要是在判决点做 ISI 抵消。

## 8. DFE 和 CDR 的关系

CDR 要解决的问题是：

```text
我应该在每个 UI 的什么位置采样？
```

DFE 要解决的问题是：

```text
在这个采样点上，如何把过去码元的拖尾减掉？
```

二者是配合关系。

经过 DFE 后，采样点附近的垂直眼高变大，判决结果更可靠。可靠的判决结果又可以帮助 CDR 判断采样点是偏早还是偏晚。

可以理解为：

```text
CDR 调整横向位置，也就是采样时间；
DFE 调整纵向电平，也就是消除采样点处的 ISI。
```

但是在高速 SerDes 中，两者会互相影响：

```text
采样点选得不好，DFE tap 也不好收敛；
DFE 抵消效果不好，CDR 的相位判断也会变差。
```

所以实际系统中，CTLE、FFE、DFE、CDR 往往是联合工作的。

---

## 9. 总结一下 DFE 的原理

DFE，也就是判决反馈均衡器，本质上是利用已经判决出来的历史数据，估计这些历史码元在当前采样点造成的拖尾干扰，并在判决前将其减掉。由于它的反馈输入是已经判决后的 0/1 或 ±1 数据，而不是带噪声的模拟波形，所以它不会像 CTLE 那样通过增强高频来同时放大高频噪声。DFE 的反馈 tap 对应的是信道主光标之后的 post-cursor ISI，因此它只能消除过去码元拖尾对当前码元的影响，不能消除 pre-cursor ISI。DFE 的非线性来源于判决器，因为它先把连续电压判决成离散数据，再用这些离散数据进行反馈抵消。DFE 不追求整个接收波形完全恢复理想形状，而是只关心最佳采样时刻处的电平是否足够干净，所以经过 DFE 后，眼图往往表现为采样中心附近张开明显，而旁边位置不一定完全张开。实际 SerDes 中，DFE 通常会和 CTLE、FFE、CDR 配合使用：CTLE/FFE 先粗略补偿信道频响和 precursor，DFE 再精确抵消 post-cursor，CDR 则根据判决结果不断调整采样时间。
