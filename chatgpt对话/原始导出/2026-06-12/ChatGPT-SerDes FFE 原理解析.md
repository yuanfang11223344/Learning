可以这样理解：**SerDes 里的 FFE 本质上是在发送端“提前把信号修一修”，让它经过低通信道之后，接收端看到的波形尽量恢复成理想波形。**

核心理解可以分成几层：

## 1. 为什么需要 FFE？

高速 SerDes 传输时，信道不是理想导线，而更像一个**低通滤波器**。

也就是说：

低频分量比较容易通过，高频分量衰减严重。

但是高速数字信号的边沿很陡，里面包含大量高频成分。高频被衰减后，接收端看到的波形就会出现：

- 上升沿、下降沿变缓；
- 当前码元拖尾到后面的码元；
- 眼图闭合；
- 产生码间串扰，也就是 ISI。

所以 FFE 的作用就是：**在发送端预先增强高频，或者削弱低频，让信号通过信道后整体频响更平坦。**

---

## 2. FFE 为什么说是 FIR 滤波器？

FFE 可以看成一个 FIR 滤波器，因为它对当前码元和前后若干个码元做加权求和：

$$
y[n] = \sum_{k=-M}^{N} c_k x[n-k]
$$

其中：

- $x[n]$：原始发送码元；
- $y[n]$：经过 FFE 预处理后的发送波形；
- $c_k$：第 $k$ 个 Tap 的系数；
- $k=0$：主抽头，也叫 main cursor；
- $k<0$：前向抽头，pre-cursor taps；
- $k>0$：后向抽头，post-cursor taps。

简单说就是：

**发送的当前码元，不再只由当前 bit 决定，而是由当前 bit 加上前后若干个 bit 共同决定。**

---

## 3. Tap 之间为什么延迟一个 UI？

这里的 UI 是 Unit Interval，也就是一个码元周期。

因为 ISI 本质上就是：

**前一个码元、后一个码元对当前采样点产生影响。**

所以 FFE 每个 Tap 间隔一个 UI，刚好对应相邻码元之间的干扰关系。

例如一个 3-tap FFE 可以理解为：

$$
y[n] = c_{-1}x[n+1] + c_0x[n] + c_1x[n-1]
$$

意思是输出信号由：

- 后一个码元；
- 当前码元；
- 前一个码元；

共同加权组成。

---

## 4. Pre-cursor 和 Post-cursor 怎么理解？

### Pre-cursor Tap：补偿当前码元之前的干扰

Pre-cursor 对应 $k<0$。

它看起来像是用了“未来码元”，但在发送端这是可以做到的，因为发送端本来就知道即将发送的数据序列。

它主要用于补偿接收端采样点之前的波形畸变。

可以理解为：

**提前对边沿做一点预修正，让当前采样点前面的干扰减小。**

---

### Post-cursor Tap：补偿当前码元之后的拖尾

Post-cursor 对应 $k>0$。

这是更常见、更直观的部分。

由于信道低通效应，当前码元的能量会拖到后面的码元里，对后续码元造成干扰。

Post-cursor Tap 就是为了抵消这种拖尾。

可以理解为：

**当前码元后面留下的尾巴太长了，所以 FFE 提前加一个反向分量把它抵消掉。**

---

## 5. 为什么主抽头为正，其他抽头多为负？

理想情况下，希望接收端在当前采样点看到的主要是当前码元，而不是其他码元。

所以：

- $c_0$ 主抽头通常为正，负责发送当前码元的主要能量；
- 其他 Tap 常常为负，用来抵消相邻码元对当前码元的影响。

比如信道会让一个“1”码元拖尾到下一个 UI，使下一个码元被抬高。  
那么 FFE 就可以在发送端给后续位置加一个反向分量，把这部分拖尾抵消掉。

因此可以把 FFE 理解成：

**主抽头负责“发送当前信号”，其他抽头负责“减掉相邻码元带来的干扰”。**

---

## 6. FFE 的时域表现是什么？

FFE 在时域上常表现为**预加重**或**去加重**。

例如发送一个跳变信号时：

原始信号可能是：

```text
0 0 0 1 1 1
```

经过 FFE 后，跳变位置可能被增强：

```text
0 0 0 1.2 0.9 1.0
```

也就是说，在边沿处给更大的能量，让高频成分更强。

经过信道低通衰减后，接收端可能刚好恢复成接近理想的：

```text
0 0 0 1 1 1
```

所以 FFE 的目标不是让发送波形看起来更“方正”，而是让它**经过信道之后更方正**。

---

## 7. 系数怎么选择？

Tap 系数的选择取决于信道特性。

如果信道高频损耗严重，就需要更强的高频补偿；  
如果后向拖尾明显，就需要更强的 post-cursor 补偿；  
如果前向干扰明显，就需要 pre-cursor 补偿。

实际 SerDes 中，Tap 系数一般通过以下方式确定：

- 厂商预设；
- 手动调参；
- 链路训练；
- 自适应算法；
- 眼图/BER 反馈优化；
- 接收端配合 CTLE、DFE 共同调整。

不同厂商的算法细节通常不会完全公开，因为这是高速 SerDes IP 的核心技术之一。

---

## 8. 一句话总结

**FFE 就是在发送端利用多个相邻码元的加权组合，提前制造一个“被预补偿过的波形”，让它经过低通信道衰减后，在接收端采样点处尽量消除码间串扰、打开眼图、降低误码率。**

更形象地说：

**信道会把信号边沿磨圆、把当前码元拖到后面；FFE 就提前把信号边沿拉尖，并加入反向补偿，把未来会出现的拖尾提前抵消掉。**


这里的 **Pre / Post 是相对于主采样点来说的时间位置**。

## 2. 为什么 $k>0$ 是 Post-cursor？

因为在公式 $x[n-k]$ 里：

当 $k=1$ 时：

$$
x[n-k] = x[n-1]
$$

也就是**前一个码元**。

前一个码元经过低通信道后，会拖尾影响到当前码元甚至后面的码元。这个拖尾出现在它自己的主峰之后，所以叫 **post-cursor ISI**。

因此，post-cursor tap 主要是用来抵消**过去码元拖到当前采样点的干扰**。

简单说：

**过去的码元影响了现在，这种拖尾叫 post-cursor。**

---

## 3. 为什么 $k<0$ 是 Pre-cursor？

当 $k=-1$ 时：

$$
x[n-k] = x[n+1]
$$

也就是**后一个码元**。

这看起来像“未来码元”，但在发送端是可以实现的，因为发送端本来就知道后面要发送什么数据，只需要整体延迟一下输出即可。

pre-cursor tap 用来补偿的是**主采样点之前的干扰分量**。

简单说：

**未来码元提前对当前采样点附近产生影响，这类在主峰前面的分量叫 pre-cursor。**

---

## 4. 抽头系数为负只是“补偿方向”，不是 Pre/Post 的定义

你说的“除了 $k=0$ 为正，其他多为负”，这是常见情况，但不是定义 Pre/Post 的原因。

系数正负表示的是：

**这个 tap 是增强还是抵消某个码元的影响。**

而 $k<0$、$k>0$ 表示的是：

**这个 tap 在主抽头前面还是后面。**

两者不是一回事。

可以理解成：

```text
k 的正负：表示时间位置
c_k 的正负：表示补偿方向
```

例如：

- $k=1$，$c_1<0$：用过去码元的反向分量抵消拖尾；
- $k=-1$，$c_{-1}<0$：用未来码元的反向分量补偿前游干扰；
- 但在某些信道、某些编码或某些厂商实现里，某些非主抽头也可能是正的。

所以不能说：

**因为系数是负的，所以它叫 pre-cursor 或 post-cursor。**

应该说：

**因为它位于主抽头之前或之后，所以叫 pre-cursor 或 post-cursor；至于系数是正是负，取决于它要补偿的信道畸变方向。**

---

## 5. 最核心的一句话

**Pre-cursor / Post-cursor 是按时间位置划分的，不是按系数正负划分的。**

更直观地说：

**主抽头 $k=0$ 是当前码元；在主抽头前面的补偿叫 pre-cursor，在主抽头后面的补偿叫 post-cursor。系数为负只是为了抵消 ISI，并不是它们名字的来源。**

```text
时间位置：     -2     -1      0      +1     +2
```

- 当前码元主能量是 $1.00$；
- 它对前 1 拍有 $0.15$ 的影响；

- 对后 1 拍有 $0.30$ 的拖尾；
- 对后 2 拍有 $0.10$ 的拖尾。

```text
x[n-1] = 1
x[n]   = 0
x[n+1] = 0

```

$$

$$

这样在发送端提前减掉这部分拖尾影响，让接收端当前采样点更接近 0。

所以可以总结为：

**ISI 影响得越远，需要的 Tap 数可能越多；某个相邻码元对当前码元干扰越大，对应 Tap 系数的绝对值通常越大。**

- 电路复杂度增加；
- 功耗增加；
- 系数训练更复杂；
- 可能放大噪声；
- 发送端摆幅被分散，主抽头幅度下降。

这些 **Tap 的数量、位置和系数大小**，主要和下面几类因素有关：

## 1. 与信道的 ISI 持续时间有关

最直接的因素是：**一个码元的能量会拖到后面几个 UI。**

如果一个码元只影响当前拍和下一拍，那么 3-tap 可能够用；  
如果一个码元会拖到后面 3～4 个 UI，那就需要更多 post-cursor taps。

例如信道冲激响应是：

```text
时间位置：    -1      0      +1     +2
响应幅度：    0.05   1.00   0.30   0.12
```

说明：

- 当前码元主能量在 $k=0$；
- 前面有一点 pre-cursor ISI：0.05；
- 后面第一拍拖尾较大：0.30；
- 后面第二拍还有拖尾：0.12。

那么 FFE 可以设计成：

```text
Tap位置：     -1      0      +1      +2
Tap类型：    Pre    Main    Post    Post
系数：      -0.05   1.00   -0.30   -0.12
```

这里的意思是：**哪个位置的 ISI 明显，就在哪个位置加对应的 tap 去补偿。**

---

## 2. 与信道高频损耗有关

SerDes 信道通常是低通特性。  
如果高频损耗越严重，边沿越慢，ISI 越严重，FFE 的补偿也要越强。

例如短板级走线，损耗较小：

```text
信道响应：    0      1.00    0.10
Tap系数：     0      1.00   -0.10
```

可能只需要一个很小的 post tap。

但是长背板、长线缆、多个连接器之后：

```text
信道响应：    0.08   1.00    0.45    0.25    0.10
Tap系数：    -0.08   1.00   -0.45   -0.25   -0.10
```

这时拖尾更长，post-cursor taps 更多，系数绝对值也更大。

所以：

**信道越差，通常需要更多 tap、更大的补偿系数。**

---

## 3. 与传输速率有关

同一条信道，在低速和高速下表现不一样。

比如一条 PCB 走线：

- 5 Gbps 时，一个 UI 很宽，信号有足够时间稳定；
- 25 Gbps 时，一个 UI 很窄，前一个码元还没稳定，下一个码元就来了。

所以高速时 ISI 更明显。

例如：

### 5 Gbps

```text
响应：      0      1.00    0.08
FFE：       0      1.00   -0.08
```

可能 2-tap 或 3-tap 就够。

### 25 Gbps

```text
响应：     0.06   1.00    0.35    0.18
FFE：     -0.06   1.00   -0.35   -0.18
```

可能就需要 pre tap 加多个 post tap。

因此：

**速率越高，UI 越短，同样的物理信道会表现出更强的 ISI。**

---

## 4. 与主要干扰来自前面还是后面有关

如果主要是前面码元拖尾影响当前码元，就需要 post-cursor taps。

例如：

```text
时间：      0      +1     +2
响应：     1.00   0.40   0.20
```

说明当前码元会拖到后面两拍。  
对应 FFE：

```text
c0 = 1.00
c1 = -0.40
c2 = -0.20
```

这就是典型的 **post-cursor 补偿**。

---

如果采样点前面也有明显干扰，就需要 pre-cursor taps。

例如：

```text
时间：     -1      0      +1
响应：     0.15   1.00   0.25
```

说明主峰前面已经有 0.15 的能量，后面还有 0.25 的拖尾。  
对应 FFE：

```text
c-1 = -0.15
c0  =  1.00
c1  = -0.25
```

这就是同时使用 **pre tap 和 post tap**。

---

## 5. 与接收端均衡能力有关

SerDes 通常不是只有发送端 FFE，还会配合接收端：

- CTLE；
- DFE；
- VGA；
- CDR；
- slicer；
- adaptive equalizer。

如果接收端 DFE 很强，发送端 FFE 的 post-cursor tap 可以不用特别多。  
因为 DFE 本身就擅长消除 post-cursor ISI。

例如：

```text
信道拖尾：      +1 = 0.35, +2 = 0.20, +3 = 0.10
```

## 6. 与发送端摆幅和功耗限制有关

FFE 的总输出幅度是有限的。  
如果非主抽头系数太大，主抽头就会被压小。

例如总幅度预算为 1：

```text
c0 = 0.8
c1 = -0.15
c2 = -0.05
```

这是比较温和的配置。

如果补偿太强：

```text
c0 = 0.5
c1 = -0.35
c2 = -0.15
```

虽然高频补偿更强，但主抽头变小，信号整体幅度下降，接收端噪声裕量可能变差。

所以 tap 系数不能无限大，要受：

- TX 输出摆幅；
- 驱动能力；
- 功耗；
- 线性度；
- EMI；
- 眼图开口；

共同限制。

## 1. 前一个码元拖尾到当前码元：为什么叫 post-cursor？

假设有三个码元：

```text
时间：        n-1        n        n+1
码元：       前一拍     当前拍     后一拍
采样点：      ↑          ↑         ↑
```

如果前一拍 $x[n-1]$ 是 1，它经过低通信道后不会马上结束，而是会拖尾到当前拍 $n$。

```text
前一拍码元的响应：

时间：        n-1        n        n+1
             主峰       拖尾      更小拖尾
幅度：        1.0       0.3       0.1
```

站在 **前一拍码元 $x[n-1]$** 的角度看：

- 它自己的主峰在 $n-1$；
- 它影响到 $n$、$n+1$；
- 这些影响发生在它主峰之后。

所以这叫 **post-cursor ISI**。

也就是说：

**前一个码元的“后游拖尾”影响了当前码元，所以要用 post-cursor tap 补偿。**

对应 FFE 里就是：

$$
c_1 x[n-1]
$$

因为 $x[n-1]$ 是过去的码元，所以对应 $k=+1$，也就是 post-cursor tap。

## 2. 采样点前面有干扰：为什么叫 pre-cursor？

现在换一个角度，看某一个码元自己的响应。

理想情况下，一个码元的能量应该集中在自己的主采样点：

```text
时间位置：    -1       0       +1
响应：        0       1.0      0
```

但实际信道中，主峰前面可能已经有一部分能量冒出来：

```text
时间位置：    -1       0       +1
响应：       0.15     1.0     0.25
```

这里：

- $0$ 是主采样点；
- $-1$ 是主采样点之前一个 UI；
- $+1$ 是主采样点之后一个 UI。

那么 $-1$ 位置的 0.15 就叫 **pre-cursor ISI**，因为它出现在主峰之前。

所以：

**pre-cursor 不是说“前一个码元”，而是说某个码元的能量出现在它主采样点之前。**

## 3. 用一个完整例子看

假设信道响应是：

```text
相对主采样点：   -1       0       +1       +2
响应幅度：       0.10    1.00    0.35    0.15
```

含义是：

```text

 0：主峰                         → main cursor
+1：主峰之后第一拍拖尾            → post-cursor
+2：主峰之后第二拍拖尾            → post-cursor
```

那么 FFE 可能设置成：

```text

类型：          Pre     Main     Post     Post
系数：         -0.10    1.00    -0.35    -0.15
```

注意：

**Pre / Post 是看这个干扰相对于主峰在前还是在后。**

不是看它来自哪个物理码元，也不是看系数正负。


所以：

**前一个码元拖尾影响当前码元，是 post-cursor；后一个码元提前影响当前码元，才是 pre-cursor。**
