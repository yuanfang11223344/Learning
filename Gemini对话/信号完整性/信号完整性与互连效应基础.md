## 一、信道衰减的原因

### Q: 发送端和接收端之间为什么会有信道衰减？即使发送完美矩形波，接收到的波形也会在边沿处变缓？

**A:** 传输路径可以看成一个**低通滤波器**——频率越高，衰减越大。

**三大物理原因：**

| 效应 | 机制 | 影响 |
|------|------|------|
| **趋肤效应** | 高频电流集中在导体表面，有效截面积缩小 → 等效电阻↑ | 高频衰减 |
| **寄生电容** | 走线与地之间存在分布电容，对高频形成低阻抗通路 | 边沿变缓 |
| **介质损耗** | PCB 基材（FR4）或芯片绝缘层对高频电磁波的吸收 | 幅度衰减 |

**芯片内部互连 vs PCB 互连：**

| | 芯片内部（stdcell→stdcell） | PCB 走线 |
|------|------|------|
| 主要矛盾 | RC 延迟 | 传输线效应 |
| 衰减原因 | 寄生 RC 充放电 | 趋肤效应 + 介质损耗 |
| 分析模型 | NLDM/CCS 查表 | S 参数 / IBIS |
| 优化手段 | 插 buffer、换线宽 | 均衡器、预加重 |

## 二、信号完整性（SI）对时序的影响

### 串扰（Crosstalk）

相邻走线之间的寄生耦合电容会导致**信号串扰**：

```
         C_couple
     ┌─────||─────┐
     │            │
  Line_A        Line_B
     │            │
     ▼            ▼
  Aggressor     Victim
```

- **同向翻转**：Victim 被"推"得更快 → 延迟减小（speed-up）
- **反向翻转**：Victim 被"拉"得更慢 → 延迟增大（slow-down）
- **Delta Delay**：这个由串扰引起的延迟变化量

STA 工具（如 PrimeTime SI）会分析串扰对时序的影响，并在 Timing Report 中标注 `XTALK` 或 `delta_delay`。

### IR Drop

芯片内部电源网络的电阻导致实际到达 cell 的 Vdd 低于标称值：

```
V_actual = V_dd_nominal - I * R_pg

IR Drop 大 → V_actual 低 → 门延迟变大 → Setup 恶化
```

- **静态 IR Drop**：平均电流 × 电源网络电阻
- **动态 IR Drop**：瞬时大电流（时钟沿大量寄存器同时翻转）造成瞬间电压塌陷

Decap（去耦电容）单元是缓解动态 IR Drop 的主要手段。

## 四、总结与举一反三

### 核心要点

1. **信道的低通特性**导致信号边沿变缓——趋肤效应、寄生电容、介质损耗。
2. **芯片内部主要矛盾是 RC 延迟**，板上才是传输线效应。
3. **串扰导致延迟变化**（delta delay），STA 需要 SI 分析。
4. **IR Drop 直接侵蚀电压余量**，恶化 setup timing。

### 举一反三

- **IBIS 模型**：IO 端口的信号完整性仿真用 IBIS（I/O Buffer Information Specification），描述 IO 的 V-I 曲线和开关特性，不暴露内部电路。
- **S 参数**：频域描述传输通道的反射、衰减、串扰，高速 SerDes 设计必用。
- **Pre-emphasis / De-emphasis**：发送端主动增强信号边沿的高频分量，对抗信道衰减。PCIe、USB 3.0、DDR5 都有此机制。
- **Decap 的局限**：Decap 单元本身有漏电（栅氧化层穿隧电流），不是越多越好；先进工艺中 MIMCAP（金属-绝缘体-金属电容）逐渐替代 MOS Decap。
